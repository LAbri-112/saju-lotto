# 사주 로또 추천기

과거 로또 6/45 당첨 번호 통계와 당첨분포를 우선 반영해 추천 조합을 만드는 브라우저 앱입니다. 연금복권720+ 탭은 로또와 별도의 입력값, 상태, 캐시, 저장키로 동작합니다. 사주와 개인 숫자는 번호 선택을 돕는 재미 요소이며, 실제 추첨 확률을 보장하지 않습니다.

## 실행

`index.html`을 브라우저에서 열면 바로 사용할 수 있습니다.

## 데이터 갱신

새 회차가 나온 뒤 아래 명령을 실행하면 로또 데이터가 갱신됩니다.

```powershell
node tools/update-lotto-data.mjs
```

연금복권720+ 데이터는 별도 명령으로 갱신합니다.

```powershell
node tools/update-pension-data.mjs
```

로또 데이터 수집은 동행복권 JSON 엔드포인트를 먼저 시도하고, 최근 회차는 동행복권 당첨결과 페이지에서 2등 당첨자 수와 당첨금도 보강합니다. 연금복권 데이터는 [동행복권 연금복권720+ 결과 페이지](https://www.dhlottery.co.kr/pt720/result)와 회차별 `gameResult.do?method=win720&Round={round}` 결과를 기준으로 가져옵니다.

데이터 파일이 비었거나 숫자 형식이 깨졌는지 확인하려면 아래 명령을 사용합니다.

```powershell
npm run validate:data -- lotto
npm run validate:data -- pension
```

## 주의

로또와 연금복권 추첨은 독립 난수 이벤트입니다. 과거 통계, 사주, 개인 숫자, 분포점수는 번호 조합을 정리하는 참고 지표일 뿐 당첨을 보장하지 않습니다. 같은 복권 안에서는 모든 가능한 번호 조합의 실제 추첨 확률이 같습니다.

## 명리 모델

명리 반영 방식은 [docs/research-notes.md](docs/research-notes.md)에 정리했습니다. 현재 버전은 천간·지지·장간·월령·십성·용신 후보를 계산하고, 그 결과를 투명한 번호-오행 매핑으로 변환합니다.

## 위치와 개운 루틴

사는 곳이나 현재 위치를 반영해 사주상 보완 방향의 판매점 검색 링크, 추천 구매 시간대, 행운 색상·의상·음식·소품을 함께 제안합니다. 실제 판매점명과 영업 여부는 지도와 동행복권 판매점 조회에서 최신 정보를 확인하는 구조입니다.

## 당첨분포 우선 추천

기본 추천 세트는 5장, 기본 사주 반영은 0%, 기본 정렬은 당첨분포 우선입니다. 앱은 단순히 높은 점수를 만드는 대신 과거 당첨번호가 실제로 가장 많이 분포한 패턴을 먼저 봅니다. 합계대, 홀짝, 저고, 번호대 분산, 끝수 다양성, 연번, 직전 회차 중복을 기준으로 “당첨번호들이 자주 머문 모양”에 가까운 조합을 우선합니다.

`당첨분포 우선 보기`를 켜면 추천 세트 개수만큼 과거 당첨 분포와 가장 가까운 후보를 우선해서 보여줍니다. 끄면 분포가 좋은 후보 중에서도 서로 너무 비슷한 조합을 줄여 다양성을 확보합니다.

## 후보망과 Recall@K

로또 6/45 전체 조합은 8,145,060개입니다. 앱은 이 전체를 화면에 펼치지 않고, 내부 후보망을 만든 뒤 핵심 후보망과 최종 구매 추천번호를 분리해서 보여줍니다.

이 앱의 목표는 후보망을 무작정 넓혀서 당첨번호가 포함된 것처럼 보이게 하는 것이 아닙니다. 핵심 후보망 수는 고정하지 않고, 전 회차 역검증으로 학습한 당첨모양과 개인 설정을 반영해 자동으로 정합니다. 목표는 가능한 적은 후보망을 유지하면서 과거 데이터 분석과 랭킹 품질을 개선해 실제 당첨 조합의 위치를 상위로 끌어올리는 것입니다.

후보망 포함 여부는 사후 점수 기준 통과와 다릅니다. 앱은 직접 점수 평가, 실제 핵심 후보망 포함, 최종 추천 포함을 분리해서 표시합니다. ExactRecall@K는 과거 회차 검증 지표이며, 미래 당첨을 보장하지 않습니다.

- `fullUniverse`: 가능한 모든 6개 번호 조합입니다.
- `recallCandidatePool`: 내부에서 검토한 넓은 후보망입니다.
- `coreCandidatePool`: 화면에 요약되는 핵심 후보망입니다.
- `finalRecommendations`: 실제로 화면에 보이는 최종 구매 추천번호입니다.

후보망 포함률은 후보망 크기 기준의 이론적 비율입니다. 최종 구매 추천번호의 결과를 보장하지 않습니다.

매주 새 당첨번호가 들어오면 `npm run optimize:recall`이 전체 회차를 다시 거꾸로 훑습니다. 각 회차 직전 데이터만 기준으로 당첨번호가 어떤 합계대, 홀짝, 저고, 번호대 분산, 직전 중복, 최근 흐름 안에 있었는지 학습하고, 이 `lotto-recall-profile`을 다음 추천 후보망 랭킹에 반영합니다. 목표는 후보망을 키우는 것이 아니라, 가능한 적은 핵심 후보망 안에 실제 당첨번호가 들어왔던 과거 패턴을 더 강하게 반영하는 것입니다.

후보망 검증은 `npm run backtest:recall`로 실행할 수 있습니다. 이 스크립트는 N회차를 평가할 때 1회차부터 N-1회차까지만 사용해서 미래 데이터 누수를 피합니다.

빠르게 방향만 확인하려면 아래처럼 적은 회차로 실행합니다.

```powershell
npm.cmd run backtest:recall -- --draws=3 --k=420,1000,3000 --quick=true
```

전체 8,145,060개 조합 기준으로 정확히 보려면 full 모드를 씁니다. 이 방식은 느릴 수 있습니다.

```powershell
npm.cmd run backtest:recall -- --draws=30 --k=420,1000,3000 --full=true
```

특정 회차의 전체 랭킹 감사는 아래처럼 실행합니다.

```powershell
npm.cmd run rank:lotto -- --draw=1223 --k=420,1000,3000,10000 --write=true
```

전체 8,145,060개 조합을 훑는 Node 도구 초안은 `npm run rank:lotto -- --verify`로 조합 수부터 확인할 수 있습니다. 실제 전체 랭킹은 브라우저가 아니라 Node 도구 쪽에서 단계적으로 확장합니다.

## 오늘의 운세

오늘, 내일, 이번 주, 이번 달 운세를 생년월일과 출생시각 기준으로 가볍게 생성합니다. 종합운, 금전운, 관계운, 일/학업운, 건강운을 보여주며, 로또 번호 선택에서는 운세보다 당첨분포 후보를 먼저 보도록 안내합니다.

## 핵심 후보망과 명예의 전당

추천 화면에는 내부에서 검토한 생성 후보망, 핵심 후보망, 최종 구매 추천번호 수가 구분되어 표시됩니다. 기본은 당첨분포에 가까우면서 서로 너무 비슷하지 않은 5장을 보여주고, `후보 랜덤 배치` 버튼을 누르면 핵심 후보망 안에서 다른 조합을 랜덤으로 볼 수 있습니다.

추천 기록 검증은 각 사용자 브라우저에 저장된 후보를 기준으로 합니다. 새 당첨번호가 들어오면 “당신의 당첨번호였던 것은...” 문구와 함께 후보군 안의 최대 일치 조합, 일치 개수, 1~5등 후보 수를 보여줍니다.

## 회차별 점수대 학습

앱은 현재 사주 설정 기준으로 과거 회차의 실제 당첨번호가 어느 분포적합 구간에 있었는지 다시 계산합니다. 최근 회차에서 자주 나온 분포 구간을 학습 중심으로 삼고, 다음 추천은 그 구간에 가까우면서 번호 신호와 사주 보정이 맞는 후보를 우선 정렬합니다. 이 기능은 당첨을 보장하지 않고, 추천이 특정 점수대에만 굳지 않도록 보정하는 장치입니다.

## 복권집 TOP 3

구매 루틴에는 오늘/내일 구매운과 추천 복권집 TOP 3가 표시됩니다. 복권집 추천은 입력한 생활권, 사주상 보완 오행, 재성 오행, 방향성, 동행복권 당첨 판매점 공개 목록 등장 여부를 함께 반영합니다.

## 연금복권720+

연금복권 탭은 로또 추천과 서로 영향을 주지 않습니다. 생년월일, 추천 세트, 사주 보정, 추천 방식은 연금복권 전용으로 저장됩니다.

- `자리 분산형`: 각 자리 숫자와 끝자리 쏠림을 줄여 보여줍니다.
- `세트형`: 같은 6자리 번호를 여러 조로 나눠 보여줍니다.
- `혼합형`: 세트형과 자리 분산형을 함께 섞습니다.
- `완전 랜덤`: 통계 점수 안의 후보군에서 무작위로 배치합니다.

연금복권 점수는 `분산점수`입니다. 자리별 숫자 분포와 반복 정도를 얼마나 고르게 정리했는지 보여주는 참고값입니다.

## 검증 명령

`npm run check`는 앱과 도구 파일의 문법을 확인합니다.

`npm run validate:data -- lotto`는 로또 데이터 파일을 검증합니다.

`npm run validate:data -- pension`은 연금복권 데이터가 채워진 뒤 통과합니다. `data/pension-results.json`의 `count`가 0이면 연금복권 데이터 수집이 아직 완료되지 않은 상태입니다.

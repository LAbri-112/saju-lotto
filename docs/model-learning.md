# 추천 모델 학습 메모

## 1224회 재현 결과

- 회차: 1224회
- 당첨번호: 9, 18, 21, 27, 44, 45
- 보너스: 28
- 재현 기준: 1223회까지의 통계, 기본 설정
- 당시 추천점수: 82.0
- 기본 추천 필터: 90점 이상
- 결과: 기본 추천 목록에는 포함되지 않음

## 왜 밀렸나

1224회 당첨조합은 `품질관문`이 높았습니다.

- 합계: 164
- 홀짝: 4:2
- 저고: 3:3
- 최근 중복: 1
- 구간 커버리지: 4
- 끝수 다양성: 6
- 품질관문: 91.9

하지만 개별 번호의 통계·사주 신호를 합친 `신호점수`가 낮았습니다.

- 신호점수: 74.3
- 최종점수: 82.0

즉, 이 회차는 “개별 번호 신호는 약하지만 조합 균형이 좋은 번호”였습니다.

## 학습 방향

한 회차 결과만 보고 90점 기준을 낮추면 과적합 위험이 큽니다. 대신 아래 방향이 더 안전합니다.

- 90점 이상 고점수 추천은 유지합니다.
- 별도 보조 슬롯으로 “균형은 좋은데 신호가 약한 역발상 후보”를 관찰합니다.
- 여러 회차가 쌓이면 고점수 후보와 역발상 후보 중 어느 쪽이 더 자주 3개 이상 적중하는지 비교합니다.

## 최근 200회 백테스트 메모

최근 200회 기준으로 실제 당첨조합이 90점 이상이었던 경우는 매우 적었습니다.

- 90점 이상: 200회 중 1회
- 85점 이상: 200회 중 46회
- 80점 이상: 200회 중 97회
- 80점 이상 또는 균형관문 관찰군: 200회 중 104회
- 과거 당첨번호 90%를 포함하려면 점수 기준을 약 66.5점까지 내려야 합니다.

따라서 “당첨번호를 매번 90점 이상 후보군에 넣기”는 현재 점수 체계에서는 현실적인 목표가 아닙니다. 90점은 최종 추천용 고점수 기준으로 유지하고, 실제 학습은 `80점 이상 + 균형관문 우수 후보`를 별도 관찰군으로 쌓는 쪽이 더 맞습니다.

목표를 이렇게 나눕니다.

- 최종 추천: 90점 이상 고점수 후보
- 학습 관찰군: 80점 이상 후보
- 역발상 관찰군: 품질관문 90점 이상이지만 신호점수가 낮은 후보
- 회차 검증: 다음 당첨번호가 어느 구간에 들어왔는지 저장

```

```

# 추천 모델 학습 메모

## 핵심 방향

최근 검증 결과를 보면 “점수가 높은 조합” 자체보다, 실제 당첨번호가 과거에 많이 분포한 모양을 먼저 보는 편이 더 자연스럽습니다. 따라서 추천의 중심을 점수대 경쟁에서 당첨분포 적합도로 옮깁니다.

## 당첨분포 기준

앱은 후보 조합을 아래 기준으로 과거 당첨번호와 비교합니다.

- 합계대
- 홀짝 비율
- 저번호/고번호 비율
- 번호대 분산
- 끝수 다양성
- 연속번호 수
- 직전 회차와 겹치는 수

분포적합 값은 예측값이 아니라, 과거 당첨번호들이 자주 보였던 형태와 얼마나 닮았는지 보는 내부 기준입니다.

## 사주 반영의 역할

사주는 추천의 중심이 아니라 보정값입니다.

- 사주 0%: 일반 통계와 당첨분포만 사용합니다.
- 사주 50%: 당첨분포를 우선 보되 개인 보완 오행 번호를 함께 봅니다.
- 사주 100%: 당첨분포 안에서 사주 보완 오행과 맞는 번호를 더 강하게 봅니다.

즉, 사주가 분포를 대체하는 것이 아니라, 분포 안에서 개인화 방향을 잡는 역할입니다.

## 자동 학습

앱은 최신 데이터가 들어올 때마다 과거 회차를 되짚어 봅니다.

- 각 회차 직전까지의 데이터만 사용해 실제 당첨번호의 분포적합 값을 다시 계산합니다.
- 최근 최대 200회에서 실제 당첨번호가 어느 분포 구간에 자주 있었는지 집계합니다.
- 가장 자주 나온 구간을 다음 추천의 학습 중심점으로 둡니다.
- 추천 정렬은 분포적합을 가장 크게 보고, 학습 중심점과의 거리, 품질 관문, 후보 다양성, 번호 신호, 사주 보정을 보조로 봅니다.

## 기본 UX

- 추천 세트 기본값은 5장입니다.
- 사주 반영 기본값은 0%입니다.
- 사주 반영은 당첨분포를 대체하지 않고, 원하는 사람만 재미로 섞는 보정값입니다.
- 오늘의 운세는 번호 추천 근거가 아니라, 구매 루틴과 기분 정리를 돕는 가벼운 읽을거리입니다.
- 추천 화면은 필터 통과 후보 수를 크게 보여주고, 기본 5장은 당첨분포에 가장 가까운 후보로 배치합니다.
- 후보 랜덤 배치는 같은 후보군 안에서 다른 번호를 둘러보는 기능이며, 기본 추천 기준을 바꾸지는 않습니다.
- 추천 기록 검증은 각 사용자 브라우저의 저장 후보를 기준으로 최대 일치 조합과 1~5등 후보 수를 보여줍니다.

## 재현 명령

```powershell
npm run learn:draw -- --draw=1226 --numbers=4,6,13,17,26,28 --bonus=41
```

최근 200회 백테스트:

```powershell
npm run backtest:model -- --from=1026 --to=1225
```

# 무료 배포 추천

## 1순위: 링크 배포 + PWA 설치

가장 돈이 안 들고 편한 방식은 정적 웹앱으로 배포하는 것이다.

- GitHub Pages: 무료 계정의 공개 저장소에서 정적 HTML, CSS, JavaScript 사이트를 호스팅할 수 있다.
- 배포 후 사용자는 링크를 열고 Chrome 메뉴에서 "홈 화면에 추가"를 누르면 앱처럼 실행할 수 있다.
- 현재 앱은 `manifest.webmanifest`와 `service-worker.js`가 있어 PWA 설치형 웹앱으로 준비되어 있다.

장점:
- 무료
- APK 설치 경고가 없다
- 링크만 공유하면 된다
- 새 버전을 올리면 사용자가 다시 APK를 설치할 필요가 없다

한계:
- iOS/Android에서 완전한 네이티브 앱은 아니고 브라우저 기반 앱이다.
- 위치 권한, 오프라인 캐시 등은 브라우저 정책을 따른다.

## 2순위: APK 직접 배포

Capacitor 같은 도구로 현재 웹앱을 Android APK로 감쌀 수 있다. APK 파일을 직접 공유하면 스토어 비용은 없지만, 사용자는 "알 수 없는 앱 설치" 경고를 거쳐야 한다.

장점:
- 파일 하나로 배포 가능
- 플레이스토어 등록비 없이 테스트 배포 가능

한계:
- 사용자에게 보안 경고가 뜬다.
- 업데이트할 때 APK를 다시 보내야 한다.
- Android Studio와 서명키 관리가 필요하다.

## 3순위: Google Play 배포

정식 스토어 배포는 신뢰도가 높지만 Google Play Console 개발자 계정 등록비가 필요하다. Google 공식 문서 기준으로 일회성 등록비는 US$25다. 개인 개발자 계정은 테스트 요구사항도 확인해야 한다.

## 현재 추천

초기 배포는 GitHub Pages 링크 + PWA 설치가 가장 좋다. 사용자가 늘고 앱 이름, 아이콘, 개인정보 처리방침, 위치 권한 설명이 정리되면 APK 또는 Play Store 배포를 검토한다.

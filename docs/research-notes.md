# 명리 기반 추천 엔진 설계 메모

이 앱은 로또 당첨확률을 예측하지 않는다. 모든 6개 번호 조합의 1등 확률은 동일하다는 전제 위에서, 사주명리학의 해석 구조를 개인화된 번호 선택 규칙으로 변환한다.

## 반영한 명리 구조

- 사주는 년주, 월주, 일주, 시주의 네 기둥으로 구성하고, 각 기둥은 천간과 지지의 조합으로 본다.
- 일간을 명식의 기준점으로 삼고, 나머지 천간과 지지의 장간을 일간과 비교해 십성을 계산한다.
- 월령은 계절적 힘으로 보아 오행 강약 판단에 더 큰 비중을 둔다.
- 지지는 표면 오행만 보지 않고 장간을 풀어 오행 분포에 반영한다.
- 오행이 한쪽으로 치우친 경우 태과·불급으로 보고, 중화에 도움이 되는 오행을 용신 후보로 둔다.
- 재성 강화형 모드에서는 일간이 극하는 오행을 금전 테마로 보고 보조 가중한다.
- 조후 균형형 모드에서는 월령의 한열조습을 보완하는 오행을 추가 가중한다.

## 번호 연결 규칙

전통 명리학에는 로또 6/45 번호와 오행을 직접 연결하는 표준 규칙이 없다. 따라서 앱은 아래의 투명한 휴리스틱을 사용한다.

- 번호대 오행: 1-10 토, 11-20 수, 21-30 화, 31-40 금, 41-45 목
- 끝수 오행: 1·6 수, 2·7 화, 3·8 목, 4·9 금, 5·0 토
- 순환 오행: 번호를 5로 나눈 흐름을 수·목·화·토·금 순서로 배치
- 음양 보정: 홀수는 양, 짝수는 음으로 보고 일간 음양과 비교

최종 사주 점수는 번호대 50%, 끝수 32%, 순환 18%의 오행 혼합값에 용신 후보 점수와 음양 보정을 적용한다. 여기에 기존 로또 통계 점수, 합계 균형, 홀짝 균형, 고저 균형, 최근 회차 중복 제한을 함께 반영한다.

## 위치와 구매 루틴

현재 버전은 판매점명을 고정 데이터로 박아두지 않는다. 판매점 영업 여부와 주변 매장 정보는 자주 바뀔 수 있으므로, 앱은 아래 방식으로 추천한다.

- 사용자가 입력한 동네명 또는 브라우저 현재 위치를 기준으로 한다.
- 용신 후보 오행의 방위를 산출해 “어느 방향의 판매점”이 좋은지 제안한다.
- 일간이 극하는 재성 오행은 금전 테마 방향으로 따로 제안한다.
- 추천 버튼은 구글지도, 네이버지도, 동행복권 판매점 조회로 연결한다.
- 구매 시간은 동행복권 로또6/45 판매시간 기준인 일~금 06:00~24:00, 토요일 06:00~20:00 안에서만 후보를 만든다.
- 시간대는 지지 시각과 오행을 연결해 용신 후보와 재성 오행에 맞는 2시간 단위 구간을 우선한다.

## 개운 아이템

색상, 의상, 음식, 소품은 용신 후보 오행을 생활 루틴으로 번역한 것이다. 이는 전통 명리학의 직접 규칙이라기보다, 오행 상징을 현대 라이프스타일 추천으로 변환한 UX 레이어다.

## 참고자료 축

- KCI 사주명리학 연구 동향 논문: 현대 사주명리 연구가 상담, 교육, 사회문화 영역으로 확장되고 있음을 정리한다.
- KCI 오행 태과·불급 논문들: 특정 오행의 과다와 부족, 중화 개념을 사례 분석한다.
- BaZi / Four Pillars 자료: 천간, 지지, 장간, 일간, 오행 상생상극, 네 기둥 구조를 정리한다.
- 공개 고전: 《淵海子平》, 《三命通會》 계열은 십성, 천간지지, 장간, 월령 해석의 고전적 배경으로 사용한다.
- 달서비 블로그의 로또 API 수집 예시: 동행복권 `common.do?method=getLottoNumber&drwNo=` JSON과 `common.do?method=main`의 `lottoDrwNo` 최신 회차 추출 방식을 데이터 갱신 도구에 참고했다.
- 동행복권 공식 사이트: 회차, 추첨일, 6개 당첨번호, 보너스 번호 필드를 패턴 분석의 기본 원천 구조로 삼는다.

## 한계

- 월주와 사주 기준 연도는 고정 날짜가 아니라 태양 황경 근사식으로 계산한 24절기 절입 시각을 기준으로 잡는다. 한국천문연구원/공공 역법 데이터와 같은 전문 역법표를 완전히 내장한 것은 아니므로 절입 경계 몇 분 단위의 최종 검증은 공식 역법표와 대조해야 한다.
- 합·충·삼합·방합·공망·천을귀인·문창귀인·도화·역마는 대표 규칙을 반영한다. 다만 학파별 세부 조건과 격국/용신 판단은 차이가 크므로, 앱은 이를 번호 개인화 보정 신호로만 사용한다.
- 대운은 성별과 태어난 해의 음양을 기준으로 순행/역행을 나누고, 출생 시각에서 다음/이전 절기까지의 거리를 3일 1년 기준으로 환산해 시작 시점을 잡는다. 현재 대운과 앞선 대운 흐름을 화면에 표시한다.
- 세운·월운·일진은 현재 날짜의 년주·월주·일주를 산출해 원국의 보완 오행과 약하게 대조한다. 추천 번호에는 통계 분포가 우선이며, 대운/세운은 사주 보정 안에서 작은 가중치로만 쓴다.
- 격국은 월령의 주된 장간을 일간 기준 십성으로 바꿔 간이 격국으로 표시한다. 이는 전문 감정의 격국 성패 판단이 아니라 앱 내 해석 설명과 용신 후보 산출을 돕는 기준이다.
- 번호와 오행의 연결은 고전의 직접 규칙이 아니라 앱의 해석 모델이다.
- 이 모델은 개인화된 선택 경험을 만드는 도구이며, 당첨 확률을 높인다는 통계적 근거는 없다.

## 당첨분포 우선 추천 설계

추천값은 당첨 가능성이 아니라 과거 당첨번호의 분포와 얼마나 닮았는지 보는 내부 기준이다. 단순히 높은 점수를 만드는 대신 실제 당첨번호가 많이 모였던 패턴을 우선한다.

- 무작위 추천만 하지 않고 수천 개의 후보 조합을 생성한다.
- 후보 조합의 합계대, 홀짝, 저고, 번호대 분산, 끝수 다양성, 연번 수, 직전 회차 중복 수를 과거 당첨번호 분포와 비교한다.
- 분포적합 값이 높은 후보를 우선 정렬한다.
- 사주 반영값은 당첨분포 안에서 개인 보완 오행과 맞는 번호를 보정하는 보조 신호로 사용한다.
- 최신 데이터가 들어올 때마다 과거 회차의 실제 당첨번호 분포적합 구간을 다시 계산하고, 자주 나온 구간을 다음 추천의 학습 중심점으로 사용한다.

## 운세 UX

오늘의 운세는 전문 감정문이 아니라 앱 내부의 간이 오행 흐름 읽기다. 생년월일, 출생시각, 오늘 날짜, 용신 후보 오행을 조합해 종합운, 금전운, 관계운, 일/학업운, 건강운을 표시한다. 로또 추천에서는 운세가 번호를 결정하지 않고, 과거 당첨분포에 가까운 후보를 먼저 보는 원칙을 유지한다.

## 사용자별 검증

추천 후보 검증은 서버 계정이나 실제 이용자 전체 랭킹이 아니라, 각 브라우저에 저장된 추천 기록을 기준으로 한다. 새 회차 데이터가 들어오면 저장 후보군 안에 실제 당첨번호가 있었는지, 최대로 많이 일치한 조합이 무엇인지, 1등부터 5등까지 해당되는 후보가 몇 개인지 계산해 보여준다.
